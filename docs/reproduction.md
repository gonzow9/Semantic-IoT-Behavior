# Reproduction Notes

Install locally:

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -e .
```

## 1. Compact MUD JSON

```bash
python -m semantic_iot_behavior.compact_mud \
  --input-dir data/mud_raw \
  --output-dir data/mud_compact
```

This writes one compact text file per MUD profile. Each non-empty line is one
ACE-like behavioral primitive.

## 2. Generate Reference Embeddings

Per-ACE BGE-M3 bank:

```bash
python -m semantic_iot_behavior.embed \
  --input-dir data/mud_compact \
  --pool per-ace \
  --model-name BAAI/bge-m3 \
  --output embeddings/bge_m3/reference_per_ace.npz
```

Whole compact profile bank:

```bash
python -m semantic_iot_behavior.embed \
  --input-dir data/mud_compact \
  --pool whole \
  --model-name BAAI/bge-m3 \
  --output embeddings/bge_m3/reference_whole_profile_compact.npz
```

Mean-pooled per-ACE device bank:

```bash
python -m semantic_iot_behavior.embed \
  --input-dir data/mud_compact \
  --pool mean-ace \
  --model-name BAAI/bge-m3 \
  --output embeddings/bge_m3/reference_mean_pool_per_ace.npz
```

## 3. Whiten Embeddings

```bash
python -m semantic_iot_behavior.whiten \
  --reference embeddings/bge_m3/reference_per_ace.npz \
  --input embeddings/bge_m3/reference_per_ace.npz \
  --output embeddings/bge_m3/reference_per_ace_whitened_k256.npz \
  --metadata results/local_whitening_metadata.json \
  --k 256
```

For query banks, keep the same `--reference` and change only `--input` and
`--output`. This preserves train/test separation by fitting whitening on the
canonical reference bank only.

The OpenAI banks in `embeddings/openai/` are included as shipped artifacts.
They use `text-embedding-3-large` with 1024 output dimensions. They are not
regenerated by the default local commands because doing so requires API
credentials and makes the quickstart network-dependent.

## 4. Exact-Overlap Smoke Check

```bash
python -m semantic_iot_behavior.score exact \
  --reference-dir data/mud_compact \
  --query-dir data/mud_compact \
  --method jaccard \
  --output results/local_exact_self.json
```

Supported exact text methods:

- `jaccard`
- `exact_hit_count`

## 5. MaxSim Scoring

MaxSim requires both reference and query per-ACE embedding banks:

```bash
python -m semantic_iot_behavior.score maxsim \
  --reference-npz embeddings/bge_m3/reference_per_ace_whitened_k256.npz \
  --query-npz path/to/query_per_ace_whitened_k256.npz \
  --output results/local_maxsim.json
```

The clean repo ships the reference banks and summary outputs. Full query
embedding sweeps were left out to keep the repository small.

## 6. Section 3 Evolving-Runtime Demo

Strict no-overlap demo:

```bash
python -m semantic_iot_behavior.runtime_behavior_demo \
  --mode strict-unseen \
  --query-size 3 \
  --episodes-per-device 3 \
  --output results/local_section3_strict_unseen_demo.json
```

Mixed partial-observation demo:

```bash
python -m semantic_iot_behavior.runtime_behavior_demo \
  --mode partial \
  --exact-count 2 \
  --unseen-count 2 \
  --episodes-per-device 3 \
  --output results/local_section3_partial_demo.json
```

These commands are a compact showcase of the Section 3 experiment logic. They
sample ACE-level runtime queries from the shipped embedding bank, remove
selected ACEs from the candidate references to simulate unseen behavior, and
report Jaccard, exact hit count, mean-pool, and MaxSim retrieval performance.
